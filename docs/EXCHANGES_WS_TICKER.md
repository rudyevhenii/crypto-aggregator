# 🌐 Технічна специфікація: Потокові дані WebSockets (Live Tickers)

Цей документ містить технічну специфікацію для інтеграції з WebSocket-каналами бірж **Binance**, **Coinbase Exchange**та
**Kraken (v2)** на основі актуальної офіційної документації. Документ призначений для точного мапінгу та десеріалізації
вхідних подій у реактивному бекенді додатка.

---

## ⏱️ Загальна специфікація протоколів та часових міток

| Біржа           | Базовий WebSocket URI                 | Тип / Формат часу         | Напрямок оновлень                     |
|:----------------|:--------------------------------------|:--------------------------|:--------------------------------------|
| **Binance**     | `wss://stream.binance.com:9443/ws`    | Мілісекунди (`Long`)      | Кожні 2000 мс (ковзне вікно)          |
| **Coinbase**    | `wss://ws-feed.exchange.coinbase.com` | ISO 8601 Рядок (`String`) | При кожній угоді (Match)              |
| **Kraken (v2)** | `wss://ws.kraken.com/v2`              | RFC3339 Рядок (`String`)  | При кожній угоді (Trades за дефолтом) |

## 1. 🟡 Binance WebSocket API

Стрім надає статистику 24-годинного ковзного вікна (rolling window) для конкретного символу (від поточного `requestTime`
до 24 годин тому). Дані оновлюються автоматично кожні 2000 мс.

* **Базовий URI з'єднання:** `wss://stream.binance.com:9443/ws`
* **Формат назви потоку (Stream Name):** `<symbol>@ticker` (строго в нижньому регістрі, наприклад: `btcusdt@ticker`).

### ⚙️ Фрейм підписки (Subscribe Request)

```json
{
  "method": "SUBSCRIBE",
  "params": [
    "btcusdt@ticker",
    "ethusdt@ticker"
  ],
  "id": 1
}
```

### 📦 Структура відповіді від сокета (24hr Ticker Event)

```json5
{
  "e": "24hrTicker",     // Тип події (Event type)
  "E": 123456789,        // ❗ Unix Timestamp події у МІЛІСЕКУНДАХ (Event time) -> ЦІЛЬОВИЙ ЧАС
  "s": "BTCUSDT",        // ❗ Код торгової пари (Symbol) -> ЦІЛЬОВА ТОРГОВА ПАРА
  "p": "0.0015",         // Зміна ціни (Price change)
  "P": "250.00",         // Відсоткова зміна ціни (Price change percent)
  "w": "0.0018",         // Середньозважена ціна (Weighted average price)
  "c": "0.0025",         // ❗ Поточна остання ціна (Last price) -> ЦІЛЬОВА ЦІНА
  "Q": "10",             // Об'єм останньої угоди (Last quantity)
  "o": "0.0010",         // Ціна відкриття (Open price)
  "h": "0.0025",         // Максимальна ціна за 24г (High price)
  "l": "0.0010",         // Мінімальна ціна за 24г (Low price)
  "v": "10000",          // Загальний об'єм торгів у базовій валюті (Total traded base asset volume)
  "q": "18",             // Загальний об'єм торгів у квотованій валюті (Total traded quote asset volume)
  "O": 0,                // Час відкриття статистики (Statistics open time)
  "C": 86400000,         // Час закриття статистики (Statistics close time)
  "F": 0,                // ID першої угоди (First trade ID)
  "L": 18150,            // ID останньої угоди (Last trade Id)
  "n": 18151,            // Загальна кількість угод (Total number of trades)
  "ps": "BTCUSDT",       // Символ пари після CM міграції (Pair symbol)
  "st": 1                // Тип символу після CM міграції (1 = UM, 2 = CM)
}
```

## 2. 🔵 Coinbase Exchange API

Канал `ticker` надає оновлення цін у реальному часі при кожному збігу ордерів (Match). Оновлення групуються (batching) у
разі каскадних збігів для оптимізації пропускної здатності.

* **Базовий URI з'єднання:** `wss://ws-feed.exchange.coinbase.com`

### ⚙️ Запит на підписку (Subscribe Request)

```json
{
  "type": "subscribe",
  "product_ids": [
    "ETH-USD",
    "BTC-USD"
  ],
  "channels": [
    "ticker"
  ]
}
```

### 📦 Структура відповіді від сокета (Ticker Message)

```json5
{
  "type": "ticker",
  "sequence": 37475248783,
  "product_id": "ETH-USD",              // ❗ Ідентифікатор торгової пари (Product ID) -> ЦІЛЬОВА ТОРГОВА ПАРА
  "price": "1285.22",                   // ❗ Поточна остання ціна (Price) -> ЦІЛЬОВА ЦІНА
  "open_24h": "1310.79",                // Ціна відкриття за останні 24 години
  "volume_24h": "245532.79269678",      // Загальний об'єм торгів за 24 години
  "low_24h": "1280.52",                 // Мінімальна ціна за 24 години
  "high_24h": "1313.8",                 // Максимальна ціна за 24 години
  "volume_30d": "9788783.60117027",     // Об'єм торгів за останні 30 днів
  "best_bid": "1285.04",                // Краща ціна купівлі (Best bid)
  "best_bid_size": "0.46688654",        // Об'єм на кращій ціні купівлі
  "best_ask": "1285.27",                // Краща ціна продажу (Best ask)
  "best_ask_size": "1.56637040",        // Об'єм на кращій ціні продажу
  "side": "buy",                        // Напрямок останньої ініціюючої угоди (buy/sell)
  "time": "2022-10-19T23:28:22.061769Z",// ❗ Точний час у форматі ISO 8601 (UTC) -> ЦІЛЬОВИЙ ЧАС
  "trade_id": 370843401,                // Глобальний ID угоди
  "last_size": "11.4396987"             // Об'єм останньої угоди
}
```

## 3. ⚪ Kraken WebSocket API (Нова версія v2)

**🔥 Важливе оновлення (v2):** На відміну від старої версії v1 (яка надсилала сирі позиційні масиви `[]`), версія *
*Kraken v2 повністю перейшла на стандартизовані JSON-об'єкти (`{}`)**. Стрім транслює дані рівня Level 1 (Top of the
book + останні трейди) і оновлюється на основі торгових подій.

* **Базовий URI з'єднання:** `wss://ws.kraken.com/v2`
* **Назва каналу:** `ticker`

### ⚙️ Запит на підписку (Subscribe Request)

```json5
{
  "method": "subscribe",
  "params": {
    "channel": "ticker",          // Цільовий канал: тикер ринку
    "symbol": [                   // Масив валютних пар (формат: базовий/квотований через слеш)
      "BTC/USD",
      "ETH/USD"
    ],
    "event_trigger": "trades",    // Тригер оновлень (trades — на кожну угоду, bbo — на зміну спреду)
    "snapshot": true,             // Запит статичного зрізу (snapshot) одразу після підписки
    "req_id": 123                 // Необов'язковий ID клієнтського запиту для верифікації
  }
}
```

### 📦 Структура відповіді від сокета (Snapshot / Update Response)

Обидва типи повідомлень (`snapshot` та `update`) мають ідентичну об'єктну структуру. Потокові оновлення приходять у полі
`data`, яке завжди містить **лише один єдиний об'єкт на першій позиції масиву**.

### 📦 Структура відповіді від сокета (Snapshot / Update Response)

```json5
{
  "channel": "ticker",      // Підтвердження каналу передачі
  "type": "update",         // Тип повідомлення (snapshot або update)
  "data": [
    {
      "symbol": "BTC/USD",                     // ❗ Код валютної пари (Symbol)
      "last": 63995.50,                        // ❗ Поточна ціна останньої угоди -> ЦІЛЬОВА ЦІНА
      "timestamp": "2022-12-25T09:30:59.123456Z", // ❗ Точний час RFC3339 -> ЦІЛЬОВИЙ ЧАС
      "ask": 64000.00,                         // Краща ціна продажу (Best ask price)
      "ask_qty": 1.500,                        // Об'єм на кращій ціні продажу
      "bid": 63990.00,                         // Краща ціна купівлі (Best bid price)
      "bid_qty": 2.000,                        // Об'єм на кращій ціні купівлі
      "change": 250.00,                        // 24-годинна зміна ціни
      "change_pct": 0.39,                      // 24-годинна зміна ціни у відсотках
      "high": 64200.00,                        // Максимальна ціна за останні 24 години
      "low": 63100.00,                         // Мінімальна ціна за останні 24 години
      "volume": 125.453,                       // 24-годинний об'єм торгів
      "vwap": 63950.00                         // Середньозважена ціна за об'ємом
    }
  ]
}
```

> 💡 **Зверніть увагу на Kraken v2:** На відміну від Binance та Coinbase, де числові значення (ціни та об'єми) летять у
> вигляді текстових рядків (`String`), Kraken v2 віддає ці поля безпосередньо як **числа з плаваючою крапкою (`Float`)**.
> У Java вашому маперу Jackson більше не потрібно робити парсинг рядка, Jackson зможе автоматично замапити ці `float` поля
> у `BigDecimal` за допомогою `BigDecimal.valueOf(node.get("last").asDouble())`. Також у v2 з'явився точний офіційний
`timestamp` у форматі RFC3339 рядка, тому системний `Instant.now()` більше не потрібен!
