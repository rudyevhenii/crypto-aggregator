# 📋 Документація API: Статистика 24-годинного ринкового зрізу (24hr Tickers)

Цей документ містить детальні технічні специфікації для отримання 24-годинної ринкової статистики (ціна, максимуми, мінімуми, об'єми та відсоток зміни) з бірж **Binance**, **Coinbase Exchange** та **Kraken**. Документ слугує довідником для інтеграції REST-запитів, які використовуються для розв'язання проблеми "холодного старту" UI перед підключенням до WebSocket.

---

## ⏱️ Шпаргалка: Особливості розрахунку та формати пар

| Біржа | Наявність готового `%` зміни | Формат торгової пари (Приклад) | Тип запиту (Ендпоінт) |
| :--- | :--- | :--- | :--- |
| **Binance** | **Є** (Поле `priceChangePercent`) | Строго разом (`BTCUSDT`) | Усі монети одразу або по одній (Query) |
| **Coinbase** | **НЕМАЄ** (Рахуємо вручну) | З дефісом (`BTC-USD`) | Тільки по одній монеті (Path Variable) |
| **Kraken** | **НЕМАЄ** (Рахуємо вручну) | Специфічний (`XXBTZUSD`, `XBTUSD`) | Усі монети одразу або по одній (Query) |

> 🧮 **Формула розрахунку відсотка зміни (для Coinbase та Kraken):**
> $$\text{priceChangePercent24h} = \left( \frac{\text{lastPrice} - \text{openPrice24h}}{\text{openPrice24h}} \right) \times 100$$
> *Примітка: під час імплементації враховуйте обробку нескінченних дробів (ділення з фіксованою точністю), щоб уникнути помилок округлення.*

## 1. 🟡 Binance API

Binance має найзручніший ендпоінт, який повертає вже повністю обчислену статистику, включаючи відсоток зміни.

* **Базовий URL:** `https://api.binance.com`
* **Ендпоінт:** `/api/v3/ticker/24hr`

### ⚙️ Параметри запиту
* `symbol` (`String`, опціонально): Код пари у верхньому регістрі (наприклад, `BTCUSDT`). Якщо не передати, біржа поверне масив усіх існуючих пар.

### 📦 Структура відповіді (JSON Object)
При запиті конкретної пари повертає об'єкт. Значення приходять як рядки (`String`).

```json5
{
  "symbol": "BTCUSDT",             // [1] Торгова пара (String)
  "priceChange": "-94.99999800",   // Абсолютна зміна ціни
  "priceChangePercent": "-1.127",  // [2] ❗ Відсоток зміни (String -> BigDecimal)
  "weightedAvgPrice": "8086.12",   // Середньозважена ціна
  "prevClosePrice": "8430.82",     // Вчорашня ціна закриття
  "lastPrice": "8335.82",          // [3] ❗ Поточна остання ціна (String -> BigDecimal)
  "lastQty": "2.122",              // Об'єм останньої угоди
  "bidPrice": "8335.81",           // Краща ціна купівлі
  "askPrice": "8335.82",           // Краща ціна продажу
  "openPrice": "8430.82",          // Ціна відкриття (24г тому)
  "highPrice": "8540.00",          // [4] ❗ Максимум за 24г (String -> BigDecimal)
  "lowPrice": "8000.00",           // [5] ❗ Мінімум за 24г (String -> BigDecimal)
  "volume": "1000.00",             // [6] ❗ Об'єм торгів у базовій валюті (String -> BigDecimal)
  "quoteVolume": "8086129.98",     // Об'єм торгів у квотованій валюті
  "openTime": 1672531200000,       // Час початку періоду (Long, Мілісекунди)
  "closeTime": 1672617600000,      // Час кінця періоду (Long, Мілісекунди)
  "count": 345                     // Кількість угод
}
```

## 2. 🔵 Coinbase Exchange API

Сервіс статистики Coinbase (Product Stats) повертає лише абсолютні числа за останні 24 години. Відсоток зміни необхідно вираховувати вручну під час мапінгу DTO.

* **Базовий URL:** `https://api.exchange.coinbase.com`
* **Ендпоінт:** `/products/{product_id}/stats`

### ⚙️ Параметри запиту
* `product_id` (`String`, обов'язково): Код продукту з дефісом (наприклад, `BTC-USD`). Передається як Path-змінна, а не Query-параметр.

### 📦 Структура відповіді (JSON Object)
Відповідь дуже лаконічна, всі значення є рядками (`String`).

```json5
{
  "open": "64000.00",     // [1] ❗ Ціна відкриття (String -> BigDecimal). Потрібна для розрахунку % зміни
  "high": "65100.00",     // [2] ❗ Максимум за 24г (String -> BigDecimal)
  "low": "63500.00",      // [3] ❗ Мінімум за 24г (String -> BigDecimal)
  "volume": "1450.50",    // [4] ❗ Об'єм торгів у базовій валюті (String -> BigDecimal)
  "last": "64500.00",     // [5] ❗ Поточна остання ціна (String -> BigDecimal)
  "volume_30day": "45000" // Об'єм торгів за 30 днів (Ігноруємо)
}

```

## 3. ⚪ Kraken API

Специфічний API, який ховає масиви показників всередині об'єкта під динамічним ключем торгової пари.

* **Базовий URL:** `https://api.kraken.com`
* **Ендпоінт:** `/0/public/Ticker`

### ⚙️ Параметри запиту
* `pair` (`String`, опціонально): Назва торгової пари (наприклад, `XBTUSD`). Рекомендується передавати для економії трафіку.

### 📦 Структура відповіді (JSON Object з масивами)
Зверніть увагу: ключі об'єктів (наприклад, `"c"`, `"h"`) містять масиви з двома значеннями. Нас цікавить **тільки перше значення** `[0]` (останнє оновлення / значення за сьогодні), друге значення — це ковзне вікно за 24 години (у деяких випадках вони ідентичні, але [0] є стандартом).

```json5
{
  "error": [],
  "result": {
    "XXBTZUSD": {         // Динамічний ключ (назва пари)
      "a": ["64000.00", "1", "1.000"],          // Ask масив
      "b": ["63990.00", "2", "2.000"],          // Bid масив
      "c": ["63995.50", "0.500"],               // [1] ❗ Last trade [0] -> Поточна остання ціна (String)
      "v": ["125.453", "130.000"],              // [2] ❗ Volume [1] -> Об'єм за останні 24г (String)
      "p": ["63950.00", "63900.00"],            // VWAP
      "t": [3456, 4000],                        // Number of trades
      "l": ["63100.00", "62500.00"],            // [3] ❗ Low [1] -> Мінімум за останні 24г (String)
      "h": ["64200.00", "64500.00"],            // [4] ❗ High [1] -> Максимум за останні 24г (String)
      "o": "63500.00"                           // [5] ❗ Open -> Ціна відкриття (String). Потрібна для % зміни
    }
  }
}
```

> ⚠️ **Деталь мапінгу Kraken:** Значення `"o"` (Open) є звичайним рядком, тоді як інші показники (`c`, `v`, `l`, `h`) є масивами рядків, з яких потрібно діставати відповідні елементи за індексом.
